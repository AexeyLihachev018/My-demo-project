# Правила работы с проектом Промгидравлика

## Структура файлов

- Каждая страница — один самодостаточный `.html` файл
- Никаких внешних библиотек, фреймворков, npm-пакетов
- CSS и JS — только внутри файла (в `<style>` и `<script>`)
- Иконки — только inline SVG, не иконочные шрифты

## Дизайн

- **Фон:** тёмный, базовый цвет `#060810` или близкий к нему
- **Текст:** светлый, основной `#f1f5f9`, второстепенный `#94a3b8`
- **Акцентный цвет:** оранжевый — `#f97316` (основной по брифу), `#ea580c` (тёмный), `#ffedd5` (светлый/фон)
  > Исключение: `index.html` использует синий акцент `#3b82f6` / `#60a5fa`. Это осознанное решение — не переделывать без явной просьбы.
- **Шрифт:** `'Arial Black', Arial, sans-serif` — применяется глобально ко всему документу
- **Обычный текст** (параграфы, подписи): `font-weight: 400`, не Arial Black
- **Компоненты:** карточки с `backdrop-filter: blur`, тени через акцентный цвет

## Адаптивность

- Все страницы должны корректно отображаться на мобильных (цель — от 320px, текущий минимум — 600px)
- Breakpoint: 900px (планшеты и меньше)
- Мобильное меню — гамбургер с оверлеем (как в index.html)
- Тапабельные элементы — минимум 44px высотой

## Текст и копирайт

- Стиль: простой, дружелюбный, без канцелярита
- Без слов: «осуществляем», «обеспечиваем», «в рамках», «в целях»
- CTA-кнопки — конкретные и короткие: «Получить КП», «Заказать звонок», «Рассчитать»
- Подзаголовки — живые, не формальные

## Обработка ошибок

- Калькуляторы: при неверных или пустых данных показывать inline-подсказку рядом с полем, не alert()
- Формы: валидировать поля до отправки, подсвечивать красным с пояснением что не так
- Если результат невозможно посчитать — показывать «—» и объяснять причину (например: «Диаметр штока не может быть больше диаметра гильзы»)
- Не блокировать пользователя — ошибка должна объяснять, что сделать дальше
- Не использовать `alert()`, `confirm()`, `prompt()` — только визуальные подсказки в интерфейсе

## Процесс работы

- Перед созданием нового файла — сначала описать структуру и содержание, дождаться подтверждения
- Перед редактированием существующего файла — сначала прочитать его
- Изменения вносить точечно, не переписывать всё подряд
- Коммитить только по явной просьбе
