## 1. ЦЕЛЬ ПРОЕКТА

### Основная цель
Конвертировать инженера или главного механика, который ищет гидроцилиндр под конкретную задачу, в заявку на КП — через немедленную техническую ценность (результат расчёта) без звонков, писем и регистрации.

### Цели калькулятора подбора гидроцилиндров
1. **Подобрать диаметр поршня** по заданному усилию и рабочему давлению (Этап 1)
2. **Рассчитать полные параметры** цилиндра: объёмы полостей, подачу насоса, расчётные усилия, время втягивания (Этап 2)
3. **Сформировать артикул** по ГОСТ для включения в техническое задание или заявку
4. **Передать результат** в форму заявки на КП одной кнопкой — без повторного ввода данных

### KPI лендинга
| Метрика | Цель |
|---------|------|
| Доля пользователей, дошедших до результата Этапа 1 | ≥ 60% |
| Доля пользователей, заполнивших Этап 2 | ≥ 35% |
| Конверсия в заявку (от уникальных посетителей) | ≥ 5% |
| Конверсия «скопировал/распечатал» → заявка | ≥ 15% |

### БЛОК 2 — КАЛЬКУЛЯТОР ГИДРОЦИЛИНДРОВ ⭐ Главный блок
**Вопрос:** «Смогу ли я рассчитать здесь то, что мне нужно?»

**Компоновка:** двухколоночная — поля слева (1fr), SVG-схема справа (295px, sticky)

#### Прогресс-бар (над этапами)
```
[● Шаг 1: Диаметр поршня]  ──────  [○ Шаг 2: Полный расчёт]
```
- Активный шаг — оранжевый, заполненный круг
- Пройденный шаг — зелёный с галочкой
- Переключается автоматически при получении результата Этапа 1

#### Этап 1: Подбор диаметра поршня
**Заголовок:** «Расчёт минимального диаметра гидроцилиндра по необходимому усилию»

**Поля ввода:**
| Поле | Placeholder | Подсказка | Единицы |
|------|-------------|-----------|---------|
| Необходимое усилие | напр. 50000 | Переключатель Н / кг | Н или кг |
| Рабочее давление | 10–35 | «Мобильная техника: 10–16 МПа. Промышленность: 20–25 МПа» | МПа |

**Результаты:**
- Минимальный диаметр поршня (мм) — вычисляется по формуле D = √(4F / πP)
- Стандартный диаметр по ГОСТ (мм) — ближайший больший из ряда: 32, 40, 50, 63, 80, 100, 125, 160, 200, 250, 320
- Предварительный артикул: ГЦ-80/50 (без хода)
- Подсказка: «Уточните ход штока и введите в Этапе 2 для полного расчёта»

#### Сводка этапа 1 (показывается в начале Этапа 2)
```
┌──────────────────────────────────────┐
│ Из Этапа 1:  Диаметр гильзы — 80 мм │
│              Давление — 16 МПа       │
│                            [Изменить]│
└──────────────────────────────────────┘
```

#### Этап 2: Полный расчёт параметров
**Заголовок:** «Расчёт параметров гидроцилиндра по его размерам»

**Поля ввода:**
| Поле | Placeholder | Подсказка | Единицы |
|------|-------------|-----------|---------|
| Диаметр гильзы | 32–320 | Перенесён из Этапа 1, можно изменить | мм |
| Диаметр штока | 20–200 | Типовое соотношение d/D ≈ 0.56–0.63 | мм |
| Ход гидроцилиндра (L) | напр. 300 | «Из чертежа механизма или ТЗ» | мм |
| Время выдвижения (t) | напр. 3 | «2–5 сек — быстрые механизмы, 5–15 сек — стандарт» | сек |
| Рабочее давление (P) | 10–35 | Перенесён из Этапа 1, можно изменить | МПа |

**Кнопка «Назад»** — возврат к Этапу 1 без потери данных

**Результаты:**
- Объём поршневой полости (л)
- Объём штоковой полости (л)
- Необходимая подача насоса (л/мин)
- Расчётное время втягивания (сек)
- Толкающее усилие (Н / кг)
- Тянущее усилие (Н / кг)
- Полный артикул: ГЦ-80/50-300

#### Блок результата (после заполнения обоих этапов)
```
┌─────────────────────────────────────────┐
│ ✓ Расчёт завершён                       │
│                                         │
│ Диаметр гильзы:     80 мм  ← оранжевый │
│ Диаметр штока:      50 мм               │
│ Ход:               300 мм               │
│ Объём поршневой:  1.51 л                │
│ Расход насоса:   30.1 л/мин             │
│ Артикул:        ГЦ-80/50-300            │
│                                         │
│ Срок изготовления: стандарт 7–10 дней   │
│                                         │
│ [Скопировать]  [Распечатать]            │
│ [Получить КП на этот цилиндр →]         │
└─────────────────────────────────────────┘
```

После нажатия «Скопировать» показывать:
> «Скопировано! Хотите КП с ценой? → Отправить заявку»

#### SVG-схема гидроцилиндра (правая колонка, sticky)
- Обозначения: D (голубой, слева), d (оранжевый, справа), L (фиолетовый, снизу)
- Легенда под схемой: D, d, L, t, P — значения обновляются при вводе в реальном времени
- На мобильном: схема над полями, не sticky
- ViewBox: 332 × 185

#### Обработка ошибок
- Пустые поля: подсветка рамкой, inline-подсказка «Введите значение»
- d ≥ D: «Диаметр штока не может быть больше или равен диаметру гильзы»
- Нулевые / отрицательные значения: «Введите положительное число»
- Результат показывает «—» пока не заполнены все обязательные поля
- Никаких `alert()` — только визуальные подсказки в интерфейсе

## 5. СТИЛЬ ДИЗАЙНА

### Настроение и образ
Тёмный, технологичный, точный. Как приборная панель промышленного оборудования. Не «красиво» — а «надёжно и профессионально». Инженер должен почувствовать: «здесь знают своё дело».

### Цветовая палитра
| Роль | Цвет | HEX |
|------|------|-----|
| Фон основной | Очень тёмный синий-чёрный | `#060810` |
| Фон поверхности | Тёмный | `#0d1120` |
| Фон карточек | Тёмный с синевой | `#111827` |
| Границы | Белый, 7% прозрачность | `rgba(255,255,255,0.07)` |
| Текст основной | Почти белый | `#f1f5f9` |
| Текст второстепенный | Серо-синий | `#94a3b8` |
| Акцент основной | Оранжевый | `#f97316` |
| Акцент тёмный | Тёмно-оранжевый | `#ea580c` |
| Акцент светлый / фон | Светло-оранжевый | `#ffedd5` |

| Голубой акцент | Для D на схеме | `#06b6d4` |

### Типографика
| Элемент | Шрифт | Размер |
|---------|-------|--------|
| H1 | Arial Black | 42–52px |
| H2 (секции) | Arial Black | 32–38px |
| H3 (карточки) | Arial Black | 20–24px |
| Кнопки CTA | Arial Black | 15–16px |
| Числа статистики | Arial Black | 32–40px |
| Параграфы, подписи | Arial, 400 | 15–16px |
| Подсказки, метки | Arial, 400 | 12–13px |
| Глобальный fallback | `'Arial Black', Arial, sans-serif` | — |

### Компоненты и эффекты
- **Карточки:** `backdrop-filter: blur(12px)` + `background: rgba(255,255,255,0.035)` + тень через оранжевый
- **Кнопка CTA:** `linear-gradient(135deg, #f97316, #ea580c)` + свечение при hover
- **Поля ввода:** тёмный фон, оранжевая рамка при фокусе, inline-подсказка при ошибке
- **Результаты:** зелёная рамка при заполнении, плавное появление
- **Hover карточек:** `transform: translateY(-4px) scale(1.01)` + усиление тени
- **Фон страницы:** радиальные градиенты (синий, фиолетовый, голубой) + сетка 60×60px

### Анимации (Intersection Observer)
- Секции появляются снизу при скролле: `opacity 0→1`, `translateY 30px→0`, `duration 0.6s`
- Числа в hero считаются от 0 при первом появлении
- Прогресс-бар плавно заполняется при переходе между этапами
- SVG-схема: обводка рисуется при первой загрузке (`stroke-dashoffset`)

## 7. ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ

### Стек
- Чистый HTML5 / CSS3 / vanilla JS
- Никаких внешних библиотек, фреймворков, npm
- CSS и JS — только внутри `.html` файла (`<style>` и `<script>`)
- Иконки — только inline SVG
- Один самодостаточный `.html` файл на страницу

### Производительность
- Нет сетевых запросов (кроме отправки формы)
- Всё работает офлайн (кроме формы)
- Загрузка < 200 КБ (без изображений)

### Формулы калькулятора
```
// Этап 1
D_min = Math.sqrt(4 * F / (Math.PI * P * 1e6)) * 1000  // мм

// Этап 2
V_p = Math.PI * (D/2000)^2 * (L/1000)          // л (поршневая)
V_s = Math.PI * ((D/2000)^2 - (d/2000)^2) * (L/1000)  // л (штоковая)
Q   = V_p / t * 60                               // л/мин
t_ret = V_s / (Q/60)                             // сек втягивания
F_p = P * 1e6 * Math.PI * (D/2000)^2            // Н толкающее
F_s = P * 1e6 * Math.PI * ((D/2000)^2 - (d/2000)^2)  // Н тянущее
```

### Артикул по ГОСТ
```
ГЦ-{D}/{d}-{L}
Пример: ГЦ-80/50-300
```

### ГОСТ ряд диаметров гильзы (мм)
`[32, 40, 50, 63, 80, 100, 125, 160, 200, 250, 320]`

## 9. ЧЕГО НЕ ДОЛЖНО БЫТЬ

- `alert()`, `confirm()`, `prompt()` — только визуальные сообщения в интерфейсе
- Канцелярит: «осуществляем», «обеспечиваем», «в рамках», «в целях»
- Форма заявки раньше, чем пользователь получил ценность от калькулятора
- Поля без подсказок о диапазонах и единицах измерения
- Внешние шрифты (Google Fonts и пр.) — только системные
- Результат, который исчезает при переходе к форме заявки
- Кнопки и ссылки меньше 44px высотой на мобильном
